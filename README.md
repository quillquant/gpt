# gpt

Local control panel for [Ollama](https://ollama.com) on a single machine: download models that fit your GPU (or Apple Silicon unified memory), run each model in an isolated `ollama serve`, and chat (with optional file attachments).

**Version:** see [`VERSION`](VERSION) (semver).

## Requirements

| | macOS | Linux |
|---|---|---|
| Hardware | Apple Silicon (Metal) | NVIDIA GPU (optional but expected) |
| Python | 3.11+ via Homebrew (`brew install python`) — system `/usr/bin/python3` is too old | 3.11+ |
| Ollama | `brew install ollama` | [ollama.com/download](https://ollama.com/download) or `./gpt install` |
| Service | `brew services` / Ollama app | `systemctl` (`ollama.service`) |
| Other | `curl`, optional [`gh`](https://cli.github.com/) | `curl`, optional `gh` |

## Quick start

```bash
./gpt install         # Ollama (if missing), web venv, models.conf, optional private GitHub repo
./gpt panel           # foreground — http://127.0.0.1:8080
# or:
./gpt panel --daemon  # background (survives the launching shell; log: /tmp/gpt-panel.log)
```

Open **http://127.0.0.1:8080** in a system browser (Safari / Chrome).

## Commands

All entrypoints go through `./gpt`:

| Command | What it does |
|--------|----------------|
| `install` | Bootstrap Ollama, Python deps, `models.conf`, optional private GitHub repo |
| `panel [--daemon\|-d]` | Start the control panel on port 8080 (`--daemon` detaches) |
| `bench [models…]` | Smoke-test APIs and cold-load/chat benchmark |
| `list` / `start` / `stop` / … | Isolated per-model servers (`models.conf`) |
| `service-start` / `service-stop` | Main Ollama service (`brew services` on macOS, `systemd` on Linux) |
| `upgrade` | Install latest Ollama (`brew upgrade ollama` on macOS) |
| `update-models` | `ollama pull` every local model |

Isolated servers are also available via `./ollama-isolated.sh` (used by the web API).

### Benchmark

With the panel running:

```bash
./gpt bench                         # defaults: gemma3:4b deepseek-r1:1.5b
./gpt bench qwen2.5:7b granite4:350m
```

Reports smoke checks plus cold load (s), chat latency (s), and tok/s per model. Models must be listed in `models.conf` and already pulled (`ollama pull …`).

## Platform notes

**macOS**

- Detects unified memory via `sysctl hw.memsize` (used as the GPU budget for `models.conf`).
- Isolated servers read models from `~/.ollama/models` (Homebrew default).
- Prefer `./gpt panel --daemon` when starting from scripts or short-lived shells.

**Linux**

- Detects VRAM via `nvidia-smi`.
- Isolated servers share the systemd model store under `/usr/share/ollama/.ollama/models` when present.
- May pin `OLLAMA_LLM_LIBRARY` to a CUDA backend when those libs are installed.

## Layout

- `web/` — FastAPI UI (`server.py` + static assets)
- `models.conf` — local model → port map (gitignored); generated by `./gpt install` from `models.json` + detected memory — see `models.conf.example`
- `models.json` — shared catalog metadata (size, category, GPU fit notes)
- `models.md` — human-readable catalog summary
- `install.sh` / `bench.sh` / `ollama-isolated.sh` — invoked by `./gpt`
