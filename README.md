# gpt

Local control panel for [Ollama](https://ollama.com) on a single machine: download models that fit your GPU, run each model in an isolated `ollama serve`, and chat (with optional file attachments).

**Version:** see [`VERSION`](VERSION) (semver).

## Requirements

- Linux with NVIDIA GPU (optional but expected)
- Python 3.12+
- [GitHub CLI](https://cli.github.com/) (`gh`) if you want the install script to create a private repo
- `curl`, `systemctl` (for the main Ollama service)

## Quick start

```bash
./gpt install    # Ollama (if missing), web venv, private GitHub repo, tag
./gpt panel      # http://127.0.0.1:8080
```

## Commands

All entrypoints go through `./gpt`:

| Command | What it does |
|--------|----------------|
| `install` | Bootstrap Ollama, Python deps, private GitHub repo |
| `panel` | Start the control panel on port 8080 |
| `bench [models…]` | Smoke-test APIs and cold-load/chat benchmark |
| `list` / `start` / `stop` / … | Isolated per-model servers (`models.conf`) |
| `service-start` / `service-stop` | Main systemd `ollama.service` |
| `upgrade` | Install latest Ollama binary |
| `update-models` | `ollama pull` every local model |

Isolated servers are also available via `./ollama-isolated.sh` (used by the web API).

## Layout

- `web/` — FastAPI UI (`server.py` + static assets)
- `models.conf` — local model → port map (gitignored); generated by `./gpt install` from `models.json` + GPU VRAM — see `models.conf.example`
- `models.json` — shared catalog metadata (size, category, GPU fit notes)
- `install.sh` / `bench.sh` — invoked by `./gpt`
